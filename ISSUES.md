# PyJa 課題・制限事項一覧

ここでは、PyJa の現在の実装における課題や既知の制限事項、将来的な改善案について記録します。

---

## 1. 複数行のラムダ式におけるインデント解析とセミコロン自動補完の制限

### 概要
現在の PyJa では、1行の単純なラムダ式や、波括弧 `{}` とセミコロン `;` を明示した複数行のラムダ式は問題なく動作します。しかし、PyJa の特徴である「インデントによるブロック表現」を複数行のラムダ式に適用すると、トランスパイルエラーが発生します。

### 再現コード（エラーになるパターン）
```java
list.forEach(x ->
    System.out.println(x)
    System.out.println(x.toLowerCase())
)
```

**トランスパイル後の出力:**
```java
list.forEach(x ->
    System.out.println(x)
    System.out.println(x.toLowerCase())
); // 継続行の最終行にのみセミコロンがつくが、各文の末尾や波括弧が不足しているためエラー
```

### 技術的な背景・原因
1. **丸括弧の継続行判定 (`accumulatedParenBalance`)**:
   `PyJaConverter.java` は、行内に閉じられていない丸括弧 `(` がある場合、その括弧が閉じられるまでのすべての行を「丸括弧内での複数行記述（継続行）」と見なします。
2. **自動補完のスキップ**:
   継続行として認識された行（`accumulatedParenBalance > 0` の行）は、インデント解析による波括弧 `{}` の補完や、行末へのセミコロン `;` の自動付与がスキップされる仕様になっています。
3. **結果**:
   ラムダ式の中で複数行の文を記述しても、それらが単一の式（Expression）として扱われ、波括弧やセミコロンが補完されないため、Javaの構文エラー（セミコロン不足、および ')' または ',' の不足）となります。

### 回避策（現状の記述方法）
複数行のラムダ式を使用する場合は、以下のように波括弧 `{}` と各文の末尾のセミコロン `;` を明示的に記述する必要があります。

```java
list.forEach(x -> {
    System.out.println(x);
    System.out.println(x.toLowerCase());
})
```

### 今後の改善案（アプローチ）
* **ラムダブロック用のパース規則の導入**:
  `->` 演算子以降のインデントの増加を検出し、丸括弧の継続行判定の例外として「ラムダブロック」を認識できるようにする。
* **トランスパイラの継続行判定の改良**:
  `accumulatedParenBalance > 0` であっても、ラムダ式の `->` の後など、特定のコンテキストにおいては行末のセミコロン補完やインデントによる波括弧補完を有効にするようなルールを `PyJaConverter.java` に追加する。
